HTML Elements Showcase – Inline vs Block

HTML elements are divided into inline and block types to control how content is structured and displayed on a webpage.
Block elements, start on a new line and take up the full width available, making them useful
for defining larger sections or containers. Inline elements do not start on a new line; they only
take up as much space as their content, making them ideal for styling or linking parts of text within a block. This
separation helps web developers organize content clearly and apply layout rules more effectively.

Block-level Element

Example:<p>This is a paragraph</p>

Explanation: The <p> tag defines a block of text
that always starts on a new line.

Example:< div > This is Generic container < /div >


Explanation: < By default, it takes up the full width and starts on a new line.
It doesn’t add any visual styling on its own — but it’s very powerful when combined with CSS and JavaScript. >

Example:< blockquote > This is< /blockquote >

Explanation: < blockquote >This tag used to display a section of text quoted from another source.
By default, browsers render it with indentation (extra space on the left and right) to highlight the quoted text.

Example:< ul > This stands for unordered list.< /ul >

Explanation: < ul >
It displays a list of items with bullets (•) by default.

Example:< ol > This stands for ordered list. < /ol >

Explanation: < ol >
It displays a list of items in a specific order, usually with numbers (1, 2, 3...) by default.

Example:< hr > This is

Explanation: < hr >stands for horizontal rule.
It is used to insert a thematic break (a horizontal line) between content.
It is a self-closing tag (no closing needed).

Example:< table > This is Table< /table >

Explanation: < table >This tag is used to create a table to organize data into rows and columns.
A table is made up of:
< table > → Defines the table:
<tr > → Table row
< th > → Table header (bold & centered by default)
< td > → Table data cell

Example:<h1-h6 > This is Heading < h1-h6>

Explanation: < h1 > is the largest and most important heading, while < h6 > is the smallest and least important.

Inline Elements

Example:< span >This is </span >

Explanation: The < span > styling small parts of text, marking inline content for scripting, adding identifiers (class, id), or for accessibility hooks.It is a tiny, flexible inline container used to wrap parts of text or other inline content.

Example:< a >This is < /a >

Explanation: The <a > It is used to create hyperlinks that connect one page to another, or link to files, email addresses, or even a specific section within the same page.

Example:< img >This is < img >

Explanation: The < > This tag is used to display images on a webpage. It is an inline element and does not need a closing tag.

Example:< abbr >This is < /abbr >

Explanation: The < abbr > It is a tag that is used to define an abbreviation or acronym. When you hover over it, the full form appears (using the title attribute). It improves accessibility and helps readers understand shortened terms.

Example:< sup >This is </sub >

Explanation: The < sub > This tag is used to display subscript text — smaller text that appears below the baseline of normal text. It is commonly used in: -Chemical formulas (H₂O, CO₂) -Mathematical expressions (x₂, a₁, nᵢ) -Footnotes or references

Example:< code >This is < code >

Explanation: The < code > It is a tag used to represent a piece of computer code. It is an inline element by default. The text inside is usually displayed in a monospace (fixed-width) font. Best used for short code snippets within a sentence or paragraph.
